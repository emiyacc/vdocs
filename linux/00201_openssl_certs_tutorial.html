常用配置参数说明 ​

  • basicConstraints: 基本约束
  • subjectAltName(SAN) : 是 SSL 标准 x509 中定义的一个扩展，配置多域名时候使用

快速生成服务端自签证书 ​

用于快速生成服务端私钥和自签证书，命令行为主

shell
openssl req -x509 -newkey rsa:2048 -nodes -keyout server.key -out server.crt -subj "/C=CN/CN=www.example.com" -config <(echo -e "[req]\ndistinguished_name=dn\nx509_extensions=v3_req\n[dn]\n[v3_req]\nbasicConstraints=critical,CA:FALSE\nsubjectAltName=DNS:*.kana.com,IP:192.168.10.10,IP:192.168.10.11") -extensions v3_req -days 3650
1

生成CA私钥和证书 ​

命令行 ​

step01 生成 ca 私钥

生成2048位 rsa 的私钥，采用 des 算法加密私钥

shell
openssl genrsa -des -out ca/ca.origin.key 2048
openssl rsa -in ca/ca.origin.key -out ca/ca.key
1
2

生成2048位不加密的 rsa 私钥

shell
openssl genrsa -out ca/ca.key 2048
1

step02 生成 ca 自签证书

shell
openssl req -new -x509 -key ca/ca.key -subj '/C=CN/O=Example/OU=example.com/CN=Private Root Certificate' -out ca/ca.crt -days 3650
1

配置文件示例 ​

step01 生成 ca 私钥

shell
openssl genrsa -out ca/ca.key 2048
1

step02 编辑配置文件

shell
cat > ca/openssl.conf << EOF
[ req ]
distinguished_name = dn
x509_extensions = v3_req
prompt = no

[ dn ]
C = CN
O = Example
OU = example.com
CN = Private Root Certificate

[ v3_req ]
subjectKeyIdentifier=hash
authorityKeyIdentifier=keyid:always,issuer
basicConstraints = critical,CA:TRUE
keyUsage = critical, digitalSignature, cRLSign, keyCertSign
EOF

签发服务端证书 ​

注：可以使用 ca 证书或者中间证书来签发服务端证书，可按实际需求使用

命令行 ​

step01 生成私钥

shell
openssl genrsa -out server.key 2048
1

step02 生成证书请求文件

shell
openssl req -new -key server.key -out server.csr -subj '/C=CN/O=kana/OU=kana.com/CN=www.kana.com'
1

step03 签发服务端证书

shell
openssl x509 -req -in server.csr -out server.crt -CA it/it.crt -CAkey it/it.key -CAcreateserial -extfile <(echo "subjectAltName=DNS:docs.kana.com,IP:192.168.10.11,IP:192.168.10.22") -days 3650
1

配置文件 ​

step01 生成私钥

shell
openssl genrsa -out server.key 2048
1

step02 编辑配置文件

shell
cat > openssl.conf << EOF
[ req ]
distinguished_name = dn
x509_extensions = v3_req
prompt = no

[ dn ]
C = CN
ST = X
L = W
O = kana
OU = kana
CN = www.kana.com

[ v3_req ]
basicConstraints = critical,CA:FALSE
keyUsage = critical, digitalSignature, keyEncipherment
subjectAltName = @alt_names

[ alt_names ]
IP.1 = 192.168.10.11
DNS.1 = www.kana.com
DNS.2 = docs.kana.com
EOF

泛域名证书 ​

⚡泛域名证书保护服务器的单个主域名和该主域名下同级别的所有子域名，例如：*.example.com，cloud.example.com均为泛域名，

⚡泛域名证书只能匹配同级别的子域名，不能跨级匹配，例如*.example.com可以匹配www.example.com和docs.example.com，但不能匹配api.prod.example.com

证书校验 ​

验证通过 ca 签发的服务端证书

shell
openssl verify -CAfile ca/ca.crt server.crt
1

验证通过中间证书签发的服务端证书，需要用到证书链

shell
openssl verify -CAfile <(cat ca/ca.crt it/it.crt) server.crt
1
